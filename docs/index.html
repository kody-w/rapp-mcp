rapp-mcp brings RAPP — the Rapid Agent Prototyping Platform — into any MCP host. Two tiny, dependency-free Model Context Protocol servers turn a folder of drop-in agents, and a whole local brainstem, into tools your AI can call. Claude Desktop, GitHub Copilot CLI, Cursor — any client that speaks MCP gets the full on-device functionality, and a path all the way up to Azure and Microsoft 365 when you're ready.

// What it is

One protocol, the whole platform

RAPP's philosophy is "engine, not experience" — infrastructure, not an app. rapp-mcp is the on-ramp: it speaks the Model Context Protocol so any AI client can join the ecosystem and use it. You get two complementary servers, and you pick one or run both.

statelesshotloadstdlib only

rapp_mcp.py — agents as tools

Point it at a folder of *_agent.py files. Each agent becomes its own MCP tool, with the agent's own JSON-Schema as the tool's input schema. Drop a new file in and it hotloads — the folder is re-scanned on every call, so there's nothing to restart.

LLM-routedmemoryself-bootstrapping

rapp_brainstem_mcp.py — the brainstem brain

Bridges a whole running brainstem. The brainstem tool routes your message through the full /chat pipeline — LLM, memory, and every local agent at once. It can even install and start the brainstem for you.

The bytes are the contract. The same *_agent.py behaves identically on every machine — local laptop, teammate's box, or a CI runner.

// The two MCP servers

Register it in your host

Both servers speak MCP over stdio. Add the snippet to your client's config, restart the client, and the tools appear. Use absolute paths to the .py files (and, for rapp_mcp.py, your agents folder).

Edit ~/Library/Application Support/Claude/claude_desktop_config.json (macOS) or %APPDATA%\Claude\claude_desktop_config.json (Windows), then restart Claude Desktop. 

{
  "mcpServers": {
    "rapp-mcp": {
      "command": "python3",
      "args": ["/abs/path/rapp-mcp/rapp_mcp.py", "/abs/path/to/agents"]
    },
    "rapp-brainstem": {
      "command": "python3",
      "args": ["/abs/path/rapp-mcp/rapp_brainstem_mcp.py"]
    }
  }
}

rapp_mcp.py tools

One MCP tool per agent in your folder. Names and input schemas come straight from each agent's metadata.

<agent><agent>…hotloaded

rapp_brainstem_mcp.py tools

Three tools that wrap a whole running brainstem:

brainstem · message, new_session
brainstem_status brainstem_bootstrap

brainstem sends a plain-English message through the brainstem's /chat; new_session (default false) keeps memory continuity across turns. Set RAPP_BRAINSTEM_URL to bridge a non-default host (defaults to http://localhost:7071).

// Write an agent.py

The whole agent contract, on one screen

An agent is a single file matching *_agent.py. It extends BasicAgent and defines three things: self.name, self.metadata (a JSON-Schema function definition that becomes the tool's input schema), and perform(**kwargs) → str. Drop it in your agents folder and it's live — no restart. 

# A minimal rapp-mcp agent. Drop any *_agent.py like this into your agents folder.
try:
    from agents.basic_agent import BasicAgent
except ImportError:
    from basic_agent import BasicAgent


class HelloAgent(BasicAgent):
    def __init__(self):
        self.name = "hello"
        self.metadata = {
            "name": self.name,
            "description": "Say hello to someone.",
            "parameters": {
                "type": "object",
                "properties": {"name": {"type": "string", "description": "Who to greet."}},
                "required": ["name"],
            },
        }
        super().__init__(name=self.name, metadata=self.metadata)

    def perform(self, **kwargs):
        return f"Hello, {kwargs.get('name', 'world')}!"

Make the description fields self-sufficient — the calling AI only ever sees your metadata, so the description is the interface. The same file works unchanged in a Tier 1 brainstem, a Tier 2 Azure Function, and here in any MCP host.

// The three tiers

Local → Azure → Microsoft 365

RAPP is three self-contained tiers that share one agent contract. You start local and advance only when you choose to. rapp-mcp plugs in at Tier 1, and the brainstem itself can automate promotion upward.

T1

Brainstem local · Flask · port 7071

A local server whose sacred endpoint is POST /chat. Its LLM routes each request through your drop-in agents, with memory and on-device context. This is where rapp-mcp connects.

$ ./start.sh
→
T2

Spinal Cord Azure Functions · Azure OpenAI

The same agent contract, running serverless. An ARM template provisions Functions + Azure OpenAI; your agents move up unchanged.

$ ./deploy.sh <rg> <loc>
→
T3

Nervous System Microsoft 365 · Copilot Studio

A managed Power Platform solution imported into a Dataverse environment, surfacing the same agents inside Teams for your whole org.

import the solution .zip

How the brainstem promotes between tiers

Promotion is just more agents. Because every tier honors the same single-file agent contract, the brainstem can carry deploy agents that drive the next tier for you — provisioning Azure via the ARM template (azuredeploy.json + ./deploy.sh) for Tier 2, or importing the Copilot Studio solution for Tier 3.

From an MCP host that's the whole loop: ask the brainstem tool to promote, it runs the deploy agent locally with your own cloud credentials, and your agents light up at the next tier — no rewrite, no lock-in. You advance when you choose to.

// Quickstart

Clone, run, register

Three minutes from zero to your AI calling RAPP tools. You need only Python 3 and an MCP host.

Clone rapp-mcp

Grab the two servers and the example agent.

git clone https://github.com/kody-w/rapp-mcp
cd rapp-mcp

Run a server to sanity-check it

Serve the bundled example agents as MCP tools (Ctrl-C to stop). No install needed — it's pure stdlib.

python3 rapp_mcp.py examples

Register it in your MCP host

Add the snippet to your client config (see The two MCP servers) using absolute paths, then restart the client. Your tools — one per agent, plus the brainstem tools — appear automatically.

Optional — bring up the brainstem brain

Register rapp_brainstem_mcp.py too, then just ask your AI to use the brainstem_bootstrap tool. It installs and starts the brainstem and waits until it's healthy. Or install it yourself with the one-liner: 

curl -fsSL https://microsoft.github.io/aibast-agents-library/install.sh | bash

Then drop your own *_agent.py files into its agents/ folder — they hotload, and they're instantly callable from your MCP host.

// Security

Local-first, your own credentials

rapp-mcp is designed to keep control on your machine. Nothing phones home; the servers are small enough to read top to bottom.

🔒

Runs on your machine

Both servers are local stdio processes spawned by your MCP host. Agent code executes on-device, against local context — not a hosted, shared cloud service.

🔑

Your own creds

The brainstem authenticates with your own GitHub Copilot login; Tier 2/3 promotion uses your own Azure and Microsoft 365 credentials. rapp-mcp ships none and stores none.

📦

Zero dependencies

Pure Python standard library — no pip install, no transitive packages, no build step. A smaller surface to audit and to trust.

👁️

Readable by design

Each server is a single short file. The MCP protocol is plain JSON-RPC over stdio — you can watch exactly what crosses the wire.

🧩

You choose what loads

Only files you place in your agents folder become tools. experimental/ and disabled/ subfolders are skipped on purpose.

⚖️

Agents run code

An agent is real Python that runs with your permissions. Treat *_agent.py files like any script — review before you drop them in, especially from others.
